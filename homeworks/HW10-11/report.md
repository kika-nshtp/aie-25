# HW10-11 – компьютерное зрение в PyTorch: CNN, transfer learning, detection

## 1. Кратко: что сделано

- **Часть A** — датасет **STL10** (10 классов, изображения 96×96 RGB). Выбран как рекомендованный вариант по умолчанию; достаточно разнообразен (самолёты, автомобили, животные, транспорт), при этом имеет официальный train/test split и умеренный размер.
- **Часть B** — датасет **Pascal VOC 2007**, трек **detection**. Выбран как универсальный вариант с богатой разметкой bounding box'ов по 20 классам. Используется готовая COCO-pretrained модель `FasterRCNN_ResNet50_FPN_V2`, сравниваются два режима: `score_threshold = 0.3` (V1) и `score_threshold = 0.7` (V2).
- В части A сравнивались: SimpleCNN без аугментаций (C1), SimpleCNN с аугментациями (C2), ResNet18 с замороженным backbone (C3) и ResNet18 с partial fine-tune `layer4+fc` (C4).
- В части B сравнивались два режима по confidence-порогу: V1 даёт больше предсказаний (выше recall, ниже precision), V2 — меньше, но точнее.

## 3. Данные

### 3.1. Часть A: классификация

- Датасет: **STL10**
- Разделение: официальный `train` (5000 изображений) → 80/20 → train/val; официальный `test` (8000 изображений)
- Базовые transforms (C1, val, test):
  ```
  ToTensor()
  Normalize(mean=(0.4467, 0.4398, 0.4066), std=(0.2603, 0.2566, 0.2713))
  ```
- Augmentation transforms (C2):
  ```
  RandomHorizontalFlip(p=0.5)
  RandomCrop(96, padding=12)
  ColorJitter(brightness=0.3, contrast=0.3, saturation=0.2)
  ToTensor()
  Normalize(STL10_MEAN, STL10_STD)
  ```
- ResNet transforms (C3/C4 train):
  ```
  Resize(256) → RandomResizedCrop(224, scale=(0.7,1.0)) → RandomHorizontalFlip
  ToTensor() → Normalize(IMAGENET_MEAN, IMAGENET_STD)
  ```
- Комментарий: STL10 содержит 10 классов объектов реального мира (airplane, bird, car, cat, deer, dog, horse, monkey, ship, truck). Изображения 96×96 RGB, более высокое разрешение, чем CIFAR-10 (32×32), что делает задачу интереснее для CNN. Малый объём обучающей выборки (4000 примеров после val-split) означает, что transfer learning особенно полезен.

### 3.2. Часть B: structured vision

- Датасет: **Pascal VOC 2007** (`image_set="val"`)
- Трек: **detection**
- Ground truth: bounding boxes из XML-аннотаций VOC (`xmin, ymin, xmax, ymax`, класс объекта)
- Предсказания: boxes + scores + labels от `FasterRCNN_ResNet50_FPN_V2` (COCO pretrained)
- Комментарий: VOC 2007 val содержит 2510 изображений с разметкой 20 классов объектов. Для валидации взята выборка первых N изображений (100 по умолчанию). Задача детекции естественна для этого датасета — именно под него создан стандарт VOC Challenge. Сопоставление VOC-классов и COCO-классов выполнено через словарь имён (аeroplane→airplane, motorbike→motorcycle и т.д.).

## 4. Часть A: модели и обучение (C1-C4)

- **C1 (simple-cnn-base)**: SimpleCNN — 4 блока `Conv2d+BatchNorm2d+ReLU+MaxPool2d` (каналы 3→32→64→128→256), затем `Flatten → Linear(9216, 512) → ReLU → Dropout(0.4) → Linear(512, 10)`. Обучение без аугментаций.
- **C2 (simple-cnn-aug)**: Та же архитектура SimpleCNN. Отличие — обучающий DataLoader использует `aug_tf` с RandomHorizontalFlip, RandomCrop и ColorJitter.
- **C3 (resnet18-head-only)**: `ResNet18` с `ResNet18_Weights.DEFAULT`. Весь backbone заморожен (`requires_grad=False`), обучается только `model.fc` (заменена на `Linear(512, 10)`).
- **C4 (resnet18-finetune)**: Та же ResNet18. Сначала 5 эпох head-only, затем размораживается `layer4` с LR=1e-4 и `fc` с LR=1e-3.

Дополнительно:
- Loss: `CrossEntropyLoss`
- Optimizer(ы): `Adam` (C1/C2: lr=1e-3, wd=1e-4; C3: lr=1e-3 для fc; C4 phase2: param_groups — layer4 lr=1e-4, fc lr=1e-3, wd=1e-4)
- Batch size: `64`
- Epochs (макс): 15 (C1/C2), 8 (C3), 5+8=13 (C4)
- Критерий выбора лучшей модели: максимальный `val_accuracy` по всем эпохам

## 5. Часть B: постановка задачи и режимы оценки (V1-V2)

### Detection track

- Модель: `FasterRCNN_ResNet50_FPN_V2` (COCO pretrained, `torchvision`)
- **V1**: `score_threshold = 0.3` — пропускаем все предсказания с уверенностью ≥0.3
- **V2**: `score_threshold = 0.7` — оставляем только высокоуверенные предсказания
- Как считался IoU: попарная матрица `box_iou_matrix([N,4], [M,4])` → жадный матчинг по убыванию score
- Как считались precision / recall:
  - Сопоставление: prediction считается TP, если `IoU ≥ 0.5` с ещё не использованным GT-box
  - `precision = TP / (TP + FP)`, `recall = TP / (TP + FN)`

## 7. Анализ

SimpleCNN на STL10 обучается с нуля на всего ~4000 изображениях (80% от 5000 обучающих). При таком объёме данных возможности CNN ограничены: модель способна выучить базовые текстуры и формы, но генерализация невысокая из-за малого числа примеров. C1 (без аугментаций) демонстрирует классическое переобучение — train_acc продолжает расти, тогда как val_acc выходит на плато.

Аугментации (C2) дают заметный эффект: RandomCrop и HFlip снижают переобучение, ColorJitter добавляет инвариантность к условиям освещения. Улучшение C2 vs C1 устойчиво, хотя не радикально — потому что при малом датасете аугментации лишь частично компенсируют нехватку данных.

Pretrained ResNet18 (C3, C4) резко улучшает качество даже с замороженным backbone. ImageNet-веса дают мощные детекторы признаков из сотен тысяч изображений, что критично для STL10 с его маленьким train-сплитом. Head-only (C3) — быстрый и надёжный baseline, но он не адаптирует backbone под распределение STL10.

Fine-tuning layer4+fc (C4) позволяет последнему блоку сети подстроиться под конкретные классы STL10, что даёт дополнительное улучшение. Двухфазная стратегия (сначала head-only, потом осторожное fine-tune) предотвращает разрушение предобученных весов: маленький LR для layer4 сохраняет полезные признаки.

В части B IoU — правильная метрика для детекции, потому что она одновременно оценивает и правильность класса (через фильтрацию меток) и точность локализации bbox. При переходе от V1 к V2 precision растёт (модель "осторожнее"), recall падает (часть объектов пропускается). Mean IoU по сопоставленным предсказаниям остаётся стабильным — это значит, что сами bbox'ы у уверенных предсказаний хорошие; проблема V1 не в качестве bbox'ов, а в ложных срабатываниях.

Наиболее показательные ошибки модели при V1: детектор находит части объектов дважды (дублирующие bbox'ы), находит объекты, не размеченные в VOC GT, или срабатывает на фоновые текстуры. Всё это исчезает при V2, но за счёт пропуска реальных объектов.

## 8. Итоговый вывод

В качестве базового конфига для классификации я бы выбрал **C4 (resnet18-finetune)**: ResNet18 с двухфазным обучением (head-only → layer4+fc) стабильно превосходит SimpleCNN и head-only, обеспечивая хорошее соотношение времени обучения и качества. При ещё меньшем датасете можно ограничиться C3.

Главное из transfer learning: претренированный backbone — это не просто «хорошая инициализация», а готовые детекторы признаков. На малых датасетах (как STL10 с 4000 примерами) разница между with/without pretrained весами принципиальна. Fine-tuning только последних слоёв — разумный компромисс между адаптацией и сохранением предобученных знаний.

Главное из detection и метрик: precision/recall при IoU-матчинге — понятный и интерпретируемый способ оценить детектор без вычисления полного mAP. Порог score — это ручка, регулирующая компромисс между полнотой и точностью; в зависимости от задачи (найти всех людей vs не сигнализировать ложно) выбирается разный режим. IoU-порог 0.5 — стандартный для VOC, он достаточно строгий, чтобы отсеивать плохие локализации, но допускает небольшие смещения.
